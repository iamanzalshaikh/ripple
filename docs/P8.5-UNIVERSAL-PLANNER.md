## What P8.5 is

**P8.5 = Universal Desktop Intelligence Foundation** — not “another planner on legacy routers.”

Ripple gets **one brain**: speech → Universal Planner → Tool Executor → tools → Windows.

Mindset: **“I need another tool”** — not another router or planner.

**Replace the old P8.5 plan.** Do not finish legacy routers and refactor later (double work).

## Non-negotiables (summary)

Full detail in DEVELOPMENT-PLAN — do not implement without reading:

- **Permission pass 1 → entity bind → permission pass 2** (P4.6 bulk-delete)
- **`confirmIfNeeded`** before high-risk execute (P4.5)
- **`undoStack.push`** before mutating ops (P4.7)
- **`actionLimiter`** — not skipped
- **L0 = existing P4** `preprocessForNlu` + regex — do not reimplement
- **ExecutionPlan** = flat sequential steps only; variable loops → P9 (§10a)
- **Planner Memory** ≠ Knowledge Graph ≠ long-term memory; invalidate stale bindings + 90-day TTL
- **Confirm deny aborts entire plan** — partial result returned (§5.1)
- **Pass-2 permission** — not filesystem-only; includes whatsapp broadcast + multi-close (§5)
- **Event bus** — P8.6+; P8.5 uses direct executor side effects (§4b.1)
- **`requires` (categories) ≠ `dependsOnTools` (tool names)** — see dev plan §4a
- **`priority` selects tools; `cost` is telemetry-only** — see dev plan §4a.2
- **L0 before Entity Resolver** — cost ladder; see dev plan §11a
- **`toolExecutor.ts` is the only module that may call tool `execute()`** — see dev plan §5
- **Executor never open-ended replans** — `replanned: false` always until P9
