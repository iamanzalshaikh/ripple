# P8.5 Planner v2 — Architecture Design

**Status:** v2.0–v2.4 **implemented** | v2.5 pending  
**Depends on:** Phase A compound gate ✅, Phase B partial plans ✅  
**Companion:** [P8.5-RUNTIME-GRAPH.md](./P8.5-RUNTIME-GRAPH.md), [P8.5-REMAINING.md](./P8.5-REMAINING.md)  
**Last updated:** 2026-07-04

---

## 1. Problem statement

Phase B added compound split, partial execution, and browser tools. The runtime is active (`phase-b compound entry`, `browser.open_workspace`, `browser.search_workspace`), but planning is still **decentralized**: multiple paths can produce or replace plans, and clause → tool mapping is **parser-driven** rather than **matrix-driven**.

Symptoms in production logs:

- `resolved=1 unresolved=1` with no guaranteed tail completion
- `missing_arg:app` when workspace intents leak into `desktop.launch_app`
- `search Arthur Ghazi Season 1` → unresolved (media search not classified)
- Shadow `legacy=desktop-fast` mismatch on commands P8.5 already owns

Planner v2 does **not** replace Phase A/B scaffolding. It inserts a **single authoritative layer** between “split utterance” and “emit `ExecutionPlan`” so every clause gets a **typed classification** and a **deterministic tool** before execution.

---

## 2. Design goals

| # | Goal |
|---|------|
| G1 | **One routing authority** — no clause reaches the executor without passing the routing matrix |
| G2 | **Deterministic classification** — same normalized clause → same `ClauseType` every time |
| G3 | **Hard workspace rules** — YouTube/Gmail/WhatsApp/etc. never map to `desktop.launch_app` |
| G4 | **Completion policy** — partial is a strategy, not a dead end; every compound has a defined outcome |
| G5 | **No full-string re-parse** — compound clauses only; preserve Phase A invariant |
| G6 | **Legacy-transparent** — shadow compares matrix output vs legacy; legacy never executes when v2 owns the command |

---

## 3. Layered architecture

```
Transcript
    │
    ▼
[L0] normalize + utterance class (atomic | compound)     ← keep Phase A
    │
    ▼
[L1] splitCompoundParts() → clauses[]                    ← keep Phase B
    │
    ▼
[L2] ClauseClassifier → ClauseRecord[]                   ← NEW (v2 core)
    │
    ▼
[L3] ToolRoutingMatrix → PlanStep[] per clause           ← NEW (v2 core)
    │
    ▼
[L4] PlanAssembler → ExecutionPlan                       ← NEW (v2 core)
    │
    ▼
[L5] CompletionPolicy → full | partial+tail | clarify    ← NEW (v2 core)
    │
    ▼
validatePlan → Tool Executor
```

**v2 inserts L2–L5.** L0/L1 stay. Existing parsers become **inputs to L2**, not direct plan builders.

## 4. Core data model

### 4.1 `ClauseRecord`

Every split clause becomes a structured record before routing:

| Field | Purpose |
|-------|---------|
| `index` | Position in compound |
| `raw` | Original clause text |
| `normalized` | After `normalizeCompoundPart` |
| `clauseType` | Enum from classifier (see §5) |
| `confidence` | 0–1 (for clarify threshold) |
| `entities` | App name, workspace id, search query, shape, etc. |
| `parseSource` | Which parser matched (audit/debug) |
| `status` | `resolved` \| `ambiguous` \| `unsupported` |

### 4.2 `ClauseType` (closed enum)

v2 uses a **small fixed set**. Parsers map *into* this enum; they do not emit plans directly.

| ClauseType | Example clauses |
|------------|-----------------|
| `APP_LAUNCH` | open chrome, launch notepad |
| `APP_FOCUS` | switch to chrome, focus paint |
| `APP_CLOSE` | close notepad |
| `WORKSPACE_OPEN` | open youtube, open gmail |
| `WEB_SEARCH` | search cats, search for react hooks |
| `MEDIA_SEARCH` | search Arthur Ghazi season 1, play trailer |
| `FILE_SEARCH` | search my resume, find latest pdf |
| `FOLDER_OPEN` | open downloads |
| `FILE_OPEN` | open report.pdf |
| `TYPE_TEXT` | type hello, write notes |
| `DRAW_SHAPE` | draw a circle |
| `MOUSE_ACTION` | click, scroll, drag |
| `SAVE_FILE` | save as notes.txt |
| `FILE_MUTATE` | delete, rename, move |
| `UNKNOWN` | no confident match |

**Key rule:** `WEB_SEARCH`, `MEDIA_SEARCH`, and `FILE_SEARCH` are **never merged**. That fixes “search Arthur Ghazi Season 1” vs “search cats” vs “search my resume”.

### 4.3 `RoutingDecision`

| Field | Purpose |
|-------|---------|
| `tool` | e.g. `browser.open_workspace` |
| `args` | Typed args (no `_desktopPayload` in v2 path) |
| `reason` | Stable reason code for logs |
| `blockedTools` | Tools explicitly forbidden for this clause (audit) |

## 5. Clause classifier (L2)

### 5.1 Pipeline order (per clause)

Classification runs in **priority order**. First confident match wins.

```
1. Normalize clause (existing normalizeCompoundPart)
2. Structural guards (empty, too short, editor-only)
3. Desktop input (type/save/calc) — existing parsers
4. App control (launch/switch/close) — parseNativeAppCommand
5. Workspace open — parseWorkspaceOpenCommand
6. Media search — NEW: season/episode/play/watch patterns
7. Web search — parseBrowserWorkspaceSearch (simple queries only)
8. File search — parseSmartSearchCommand (index-backed)
9. File/folder open — parseDesktopCommand / file ops
10. Mouse/draw — pattern matchers
11. UNKNOWN
```

### 5.2 Disambiguation rules (hard)

| If clause contains… | Force type | Never |
|---------------------|------------|-------|
| `season`, `episode`, `trailer`, `watch`, `play` (media context) | `MEDIA_SEARCH` | `WEB_SEARCH`, `FILE_SEARCH` |
| `resume`, `pdf`, `document`, `download`, `folder` | `FILE_SEARCH` | `WEB_SEARCH` |
| `youtube`, `gmail`, `whatsapp` as **open target** | `WORKSPACE_OPEN` | `APP_LAUNCH` |
| `chrome`, `notepad`, `paint` as **open target** | `APP_LAUNCH` or `APP_FOCUS` | `WORKSPACE_OPEN` |
| Bare `search X` (no file/media signals) | `WEB_SEARCH` | `FILE_SEARCH` |
| Prior clause was `WORKSPACE_OPEN: youtube` + `search X` | `MEDIA_SEARCH` (context boost) | `WEB_SEARCH` |

### 5.3 Context-aware classification (compounds only)

Classifier receives **prior resolved clauses** in the same utterance:

| Prior context | Boost for next clause |
|---------------|----------------------|
| `WORKSPACE_OPEN: youtube` | `search …` → prefer `MEDIA_SEARCH` |
| `APP_FOCUS: chrome` | `search …` → `WEB_SEARCH` or `MEDIA_SEARCH` by media signals |
| `APP_LAUNCH: paint` | `draw …` → `DRAW_SHAPE` |
| `FILE_OPEN: resume.pdf` | `send …` → adapter path (out of v2 scope) |

This fixes `Open YouTube and search Arthur Ghazi Season 1` without treating it like `search cats`.

## 6. Tool routing matrix (L3)

**One row per `ClauseType`.** No exceptions outside the matrix.

| ClauseType | Tool | Forbidden |
|------------|------|-----------|
| `APP_LAUNCH` | `desktop.launch_app` | `browser.*` |
| `APP_FOCUS` | `desktop.focus_window` | `browser.*`, `desktop.launch_app` |
| `APP_CLOSE` | `desktop.close_window` | — |
| `WORKSPACE_OPEN` | `browser.open_workspace` | `desktop.launch_app` |
| `WEB_SEARCH` | `browser.search_workspace` | `desktop.launch_app`, `memory.search` |
| `MEDIA_SEARCH` | `browser.search_workspace` (YouTube URL builder) | `desktop.launch_app` |
| `FILE_SEARCH` | `memory.search` (or clarify until tool exists) | `browser.search_workspace` |
| `FOLDER_OPEN` / `FILE_OPEN` | `filesystem.open` | `desktop.launch_app` |
| `TYPE_TEXT` | `desktop.type_text` | — |
| `DRAW_SHAPE` | `desktop.mouse_move` + `desktop.mouse_drag` | — |
| `SAVE_FILE` | `desktop.save_file` | — |
| `FILE_MUTATE` | `filesystem.*` | — |
| `UNKNOWN` | — | all execution tools |

### 6.1 Workspace registry (authoritative)

Builtin workspaces (YouTube, Gmail, WhatsApp, Notion, etc.) are **`WORKSPACE_OPEN` only**. They never enter `resolveNativeApp()`.

### 6.2 Routing invariants (enforced at assemble time)

- Plan must not contain `desktop.launch_app` with workspace ids
- Every step must have required args (`app` or `url` or `query`)
- Violation → `status: ambiguous` → clarify, never execute with `missing_arg`

## 7. Plan assembler (L4)

### 7.1 Full plan

All clauses `resolved` → single `ExecutionPlan` with ordered steps.

```
"Switch to Chrome and open YouTube"
→ [focus_window(chrome), open_workspace(youtube)]
```

### 7.2 Partial plan (Phase B compatible)

Some clauses `resolved`, some `unsupported`:

- Emit `kind: partial` with resolved steps
- Attach `unresolvedClauses[]` with `ClauseRecord` metadata (not raw strings only)

### 7.3 Clarify plan

Any clause `ambiguous` OR confidence below threshold → `kind: clarify` with specific question per clause type.

---

## 8. Completion policy (L5)

Today: partial executes step 1, then tail planner succeeds or clarifies.  
v2 defines **explicit outcomes** for every compound:

| Situation | Policy | User outcome |
|-----------|--------|----------------|
| All resolved | `EXECUTE_FULL` | Run all steps |
| Resolved + tail plannable | `EXECUTE_PARTIAL_THEN_TAIL` | Run resolved, settle, run tail (Phase B today) |
| Resolved + `MEDIA_SEARCH` tail | `EXECUTE_PARTIAL_THEN_MEDIA` | Open workspace if needed, then YouTube search URL in active tab |
| Resolved + `UNKNOWN` tail | `CLARIFY_TAIL` | “I opened X but couldn’t understand Y” |
| All unsupported | `CLARIFY_ALL` | Rephrase question |
| Ambiguous clause | `CLARIFY_AMBIGUOUS` | Targeted question |

### 8.1 Tail resolver chain (ordered)

For each unresolved clause, try in order:

1. **Matrix re-route** (classifier retry with compound context)
2. **Tail-specific planner** (draw, workspace open, web/media search)
3. **Structured GPT** (clause + prior steps only — not full utterance)
4. **Clarify** (last resort)

**Rule:** Never stop at `resolved=1 unresolved=1` without picking one of the above.

### 8.2 Media search tail

`MEDIA_SEARCH` after `WORKSPACE_OPEN: youtube`:

- Do not use Google search URL
- Use YouTube search URL (`search_query=…`) in active tab via tab resolver
- If workspace not yet open, prepend `browser.open_workspace(youtube)` automatically

## 9. Entry path unification

v2 planning must be the **only** plan builder for P8.5 desktop commands.

| Entry path today | v2 rule |
|------------------|---------|
| `runAtomicPlanner` strict → `planFromNativeIntent` | Route through L2–L4 |
| `buildDesktopCommandResult` → `_desktopPayload` | Convert payload → `ClauseRecord` → matrix (no direct launch) |
| `tryL0CompoundPlan` per-clause parsers | Replace with classifier + matrix |
| `compoundUnresolvedPlanner` tail | Becomes tail step in CompletionPolicy §8.1 |
| `youtube-local` / `desktop-fast` orchestrator | Never run if v2 `shouldOwn(command)` |
| Async GPT tiers | Receive `CompoundPlanningContext`, not raw string |

### 9.1 `shouldOwn(command)` (single gate)

True when utterance is atomic or compound desktop command that L2 can attempt. When true:

- Block legacy routers (`p85LegacyGate` today; extended in v2)
- Suppress shadow mismatch logs for expected tool upgrades
- Force executor path (no payload bridge for workspace opens)

## 10. Shadow & observability

### 10.1 Logs (required per command)

```text
[planner-v2] classify utterance=compound clauses=2
[planner-v2] clause[0] type=APP_FOCUS tool=desktop.focus_window conf=0.95
[planner-v2] clause[1] type=WORKSPACE_OPEN tool=browser.open_workspace conf=0.93
[planner-v2] outcome=EXECUTE_FULL steps=2
```

### 10.2 Shadow parity v2

Compare **matrix signatures**, not legacy NOOP payloads:

```text
focus_window:switch_app:chrome | open_workspace:youtube
```

Legacy desktop-fast is **reference only**, not a competing router.

## 13. What v2 explicitly does NOT do

- Replace voice STT or normalization
- Replace focus/tab resolver implementation (uses it)
- Add new adapters (WhatsApp send, Gmail compose) — matrix slots reserved
- Delete legacy orchestrator blocks (PR4+; v2 only gates them)
- Open-ended GPT replanning (P9)

---

## 14. Summary

| Today (Phase B) | Planner v2 |
|-----------------|------------|
| Parsers → plan steps directly | Parsers → **ClauseType** → **matrix** → steps |
| Partial stops when tail unknown | **Completion policy** always picks an outcome |
| Web vs media vs file search blurred | **Three search types** with hard rules |
| Workspace sometimes hits `launch_app` | **Registry = browser only** |
| Legacy leaks on defer | **`shouldOwn`** blocks legacy |

**Planner v2 is the correctness layer:** not more features, but one deterministic path from clause to tool with a guaranteed completion decision.
