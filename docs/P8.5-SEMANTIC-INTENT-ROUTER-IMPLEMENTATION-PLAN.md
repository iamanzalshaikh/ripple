## 0. Why this exists

Command-mode failures observed live (Ctrl+Space), not P7 dictation:

| Utterance | Failure |
|-----------|---------|
| Create a roadmap to complete this project | `SHOW_SUGGESTIONS` / not found |
| Compare my implementation with production standards | GPT `none` / suggestions |
| Find missing requirements in my current project | NLU `Find→Open` → `open_project` → `project_not_found` |
| Perform a security review | Fake success via scan+analyze+typecheck+lint |
| Analyze outdated/risky dependencies | Same fake developer workflow |
| Check issue, summarize, make a note in Notepad | Partial automation; summarize/write dropped |
| Bro check my code | Needs `CODE_ANALYSIS` structured intent |

Root cause: `/commands/desktop-intent` is file/app-centric (`open_file` / `open_app` / `none`) and does not understand project-analysis or multi-step meaning.

| Layer | Owns | Does not own |
|-------|------|--------------|
| **P7 dictation** | Speak → correct → type | Planner / Semantic |

| **Planner / compound** | Steps, sticky compound, safety | Verb semantics / slang brain |

## 2. Target architecture

```text
VoiceTranscript
  → DictationMode? (Alt+Shift+Space) → P7 Speak/Correct/Type  [bypass Semantic]
  → DeterministicLocalCommand? → existing L0 planners         [bypass Semantic]
  → SemanticIntentRouter
       → confidence > 0.75  → PlannerV2 tool steps
       → 0.4–0.75           → AskConfirm
       → < 0.4              → AskClarify
  → PlanValidator → ToolExecutor → Observe/Recover
```

Logs: `[ripple-intent] raw=… intent=… confidence=… target=…`

## 3. Intent contract

```json
{
  "intent": "CODE_ANALYSIS",
  "target": "CURRENT_PROJECT",
  "action": "ANALYZE",
  "confidence": 0.92,
  "constraints": [],
  "notes": ""
}
```

### First-ship intent families

| Intent | Meaning |
|--------|---------|
| `CODE_ANALYSIS` | Bro check my code / find bugs / analyze codebase |
| `SECURITY_REVIEW` | Security-oriented review — **not** lint-as-security |
| `DEPENDENCY_AUDIT` | Outdated / risky deps → npm outdated + audit |
| `REQUIREMENTS_GAP` | Missing requirements — **never** `open_project` |
| `PROJECT_ROADMAP` | Roadmap to complete project |
| `COMPARE_TO_STANDARD` | Compare implementation vs production standards |
| `COMPOUND_WORKFLOW` | Multi-clause analysis → summarize → Notepad note |
| `NONE` | Not semantic; fall through |

Confidence gates: `>0.75` execute · `0.4–0.75` confirm · `<0.4` clarify.

## 4. Intent → tools (reuse P5)

| Intent | Plan shape |
|--------|------------|
| `CODE_ANALYSIS` | `scan_project` + `analyze_codebase` + `typecheck` (+ lint if configured) |
| `SECURITY_REVIEW` | analyze/find_code + `ai.reason_about_task` (security goal) — no fake lint success |
| `DEPENDENCY_AUDIT` | `automation.run_command` `npm outdated` + `npm audit` at inherited project root |
| `REQUIREMENTS_GAP` | analyze docs/plan signals + `ai.reason_about_task` — never rewrite to `open_project` |
| `PROJECT_ROADMAP` | `ai.reason_about_task` / `ai.generate_action_plan` then Notepad type or markdown note |
| `COMPARE_TO_STANDARD` | analyze + `ai.reason_about_task` comparison report |
| `COMPOUND_WORKFLOW` | Ordered shared-root steps: inspect → summarize → launch Notepad → type note |

Hard rule (P5): `ai.generate_action_plan` drafts only — always re-enter validator/executor.

## 5. Pipeline insertion

Wire into `tryP85FastPath` / `plannerPipeline`:

1. Keep deterministic local commands first (open Chrome, close Spotify, WhatsApp/Gmail adapters).
2. Bypass Semantic for P7 dictation and adapter-owned utterances.
3. Call Semantic **before** compound sticky terminates as `compound_unresolved`.
4. Stop treating broad “review/analyze/compare” as automatic generic developer workflow unless intent is `CODE_ANALYSIS`.

Key files:

- `electron/agent/planner/semanticIntentRouter.ts` *(new)*
- `electron/services/commandOrchestrator.ts`
- `electron/agent/planner/plannerPipeline.ts`
- `electron/agent/planner/compoundGate.ts`
- `electron/agent/planner/developerWorkflowPlanner.ts`
- `electron/automation/voice/nlu/normalizeIntent.ts`
- `electron/automation/voice/nlu/codeAnalysisIntent.ts`

## 6. NLU safety (bundled)

- Stop `Find …` → `Open my …` for requirements / dependency / security / roadmap / compare phrases.
- Prefer **raw speech** for Semantic when NLU rewrite changes verb semantics.

| # | Command | Expect |
|---|---------|--------|
| 1 | Create a roadmap to complete this project | Roadmap plan/result — not `not_found` |
| 2 | Compare my implementation with production standards | Comparison report plan — not suggestions |
| 3 | Find missing requirements in my current project | Gap analysis — not `open_project` |

| 6 | Check issue, summarize it, make a note in Notepad | Full compound completion (all clauses) |
| 7 | Bro check my code | `CODE_ANALYSIS` |
| 8 | Open Chrome / WhatsApp compose dictation | Still bypass Semantic |

Also run targeted P5/P6/P7 regressions after Semantic lands. Do **not** gate on the whole suite until the known 38 TS test-type errors are quarantined separately.

- Whisper / sidecar / streaming STT
