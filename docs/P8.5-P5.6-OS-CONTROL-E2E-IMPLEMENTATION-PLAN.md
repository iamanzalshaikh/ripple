## 2. Intent Contract layer (NEW — prevents tool chaos)

**Do not go:** utterance → random tool.

**Do go:**

```text
utterance
   → Intent Object (JSON)
   → Intent Validator
   → Entity Resolver (slots → paths/apps)
   → Safety
   → Tool(s)
   → Verification
   → Execution Ledger
```

### Example

User: `"Chuck demo onto the Desktop"`

```json
{
  "domain": "filesystem",
  "action": "copy",
  "source": { "type": "file", "name": "demo", "location": "Downloads" },
  "destination": { "location": "Desktop" },
  "confidence": 0.91,
  "producer": "semantic_os" | "l0",
  "utterance": "Chuck demo onto the Desktop"
}
```

Then: `validate(intent)` → resolve slots → `filesystem.copy_file` → verify file exists → ledger SUCCESS.

## 3. Routing policy (production — not “AI guess after failure”)

```text
utterance
   │
   ▼
L0 → Intent Object
   │
   ├─ confidence ≥ 0.90  ──► validate → execute → VERIFY → ledger
   │
   └─ else
         │
         ▼
      Semantic OS → Intent Object
         │
         ├─ confidence ≥ 0.75  ──► validate → execute → VERIFY → ledger
         ├─ 0.40–0.75          ──► confirm with user
         └─ < 0.40             ──► clarify (never run tools)
```

**Illegal:**

```text
L0 failed → GPT/automation.run_command free-guess → SUCCESS
```

Dangerous automation. Semantic OS must produce a **typed Intent**; validator rejects incomplete slots.

## 5. Execution Ledger + mandatory verification

**Not just console logs.** Every OS command writes a ledger record:

```json
{
  "command_id": "…",
  "intent": { "domain": "filesystem", "action": "copy", "…": "…" },
  "plan": [
    "resolve source demo",
    "resolve dest Desktop/Test2",
    "filesystem.copy_file"
  ],
  "execution": { "ok": true, "tool": "filesystem.copy_file" },
  "verification": {
    "file_exists": true,
    "hash_match": true,
    "path": "C:\\Users\\…\\Desktop\\Test 2\\demo.txt"
  },
  "undo": { "available": true, "kind": "delete_copy" },
  "status": "SUCCESS"
}
```

**Rule:** `status=SUCCESS` **only if** `verification` passes.

| Action | Minimum verification |
|--------|----------------------|
| copy / move / create | path exists (+ size/hash when applicable) |
| delete | path gone or in recycle |
| rename | new path exists, old gone |
| launch app | process or window visible |
| run_as_admin | elevated process evidence / UAC path — not bare exit=0 |
| kill | process absent |
| compare | report object returned |
| screenshot | file exists on disk |

This kills FEATURE_GAPS “SUCCESS but nothing happened.”

Store under: `.ripple/executions/` (project) and/or Ripple user data for global OS ops. Bounded retention.
