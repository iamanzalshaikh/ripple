## 0. Read this first

E4 is **not broken at the planner layer**. Clause split, `DRAW_SHAPE`, `PAINT_OP`, and tool routing work for most basic compounds. The production gap is:

> **Execution reports success before OS state is proven.**

This plan fixes E4 by adding **hard focus barriers**, **post-step state verification**, **complete label automation**, and **stronger OS tests** — without redesigning Planner v2 or building a Canva-style canvas engine.

**Scope:** Windows + MS Paint + Planner v2 + `ripple-native` sidecar.  
**Out of scope:** Mac/Linux Paint automation, semantic shape CV, owned design renderer.

- `focus-lock soft-continue input_not_ready` — focus check warns but step still runs

- `desktop intent: item:paint… (NLU)` — compound fell through to legacy NLU instead of Planner v2
- `shadow mismatch legacy=desktop-fast step_count_mismatch` — dual planner paths disagree

## 2. Target architecture

### 2.1 Execution pipeline (after this plan)

```text
Command
  → Planner v2 (compound gate — no NLU steal for E4)
  → Tool matrix → ExecutionPlan
  → FOR EACH step:
      1. Focus barrier (hard — block until target hwnd stable ≥300ms)
      2. Execute OS action (UIA / SendInput / mouse)
      3. State verifier (pixels / hwnd / clipboard / file)
      4. Retry once on verify fail (same step)
      5. Mark step failed if still bad → stop or partial clarify
  → Final state contract check (test / telemetry)
```

### 2.2 Three verification layers

| Layer | Question | Used when |
|-------|----------|-----------|
| **A. Action** | Did the tool run without throw? | Always (existing) |
| **B. State** | Is OS state correct now? | erase, clear, close, switch, save, clip |
| **C. Semantic** | Is the shape geometrically correct? | Future — not Phase 1 |

Phase 1 implements **A + B**. Phase 3 may add **C** (optional).

## 7. Risks and mitigations

| Risk | Mitigation |
|------|------------|
| Paint UI changes between Win10/Win11 | UIA names + coordinate fallback (already pattern) |
| Pixel verify flaky on HDR/scaling | Capture relative to window rect; threshold tuning |
| Focus barrier slows compounds | Cap wait at 3s; fail fast with clarify |
| Label text position varies | Verify text exists via UIA or OCR later, not exact coords |
| Legacy NLU still competes | Compound gate must return before desktop-fast |

## Appendix B — State contract reference

| verify mode | Pass condition |
|-------------|----------------|
| `paint_shape` | dark ≥ 60 AND quads ≥ 3 (or dark &gt; 200) |
| `paint_ink` | dark ≥ 60 (any ink including line) |
| `paint_empty` | dark &lt; 50 after step |
| `paint_diff` | diffRatio vs baseline ≥ 0.15 |
| `paint_closed` | no mspaint window with handle |
| `chrome_foreground` | foreground title matches Chrome |
| `clipboard` | regex match on clipboard text |
| `file` | file exists in Downloads with expected content |
